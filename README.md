# MG-AFSS

Release candidate for **Maturity-Gated Anti-Forgetting Sampling (MG-AFSS)**, a maturity-aware activation mechanism for AFSS-based training acceleration in remote-sensing object detection.

This package supports the manuscript:

> Maturity-Gated Anti-Forgetting Sampling for Remote-Sensing Object Detection Training

## What Is Included

- `mg_afss/run_mg_afss_train.py`: unified training entry for standard training, fixed AFSS, and MG-AFSS.
- `mg_afss/paths.py`: path helper using environment-variable overrides.
- `ultralytics_patch/`: AFSS scheduler, image-state logic, HBB/OBB trainer integration, and config patch notes for Ultralytics.
- `tools/`: environment check, dataset-yaml helper, result collection, and resolution audit utilities.
- `tests/`: focused AFSS tests copied from the development fork.

## What Is Not Included

- Raw third-party datasets.
- Pretrained or trained model weights.
- Full experiment output folders.
- Paper figures and processed numerical results.
- Generated logs and qualitative detection examples.
- Local development archives and exploratory runs.
- A full Ultralytics source tree.

## Quick Setup

1. Create a Python environment.

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

On Windows PowerShell:

```powershell
python -m venv .venv
.\\.venv\\Scripts\\Activate.ps1
pip install -r requirements.txt
```

2. Install PyTorch for your CUDA platform. The manuscript experiments used PyTorch 2.5.1+cu121 and CUDA 12.1.

3. Prepare a local Ultralytics checkout and apply the files under `ultralytics_patch/` to the matching paths in that checkout.

4. Point this package to the patched runtime:

```bash
export ULTRALYTICS_PATH=/path/to/patched/ultralytics-repo
```

On Windows PowerShell:

```powershell
$env:ULTRALYTICS_PATH="<path-to-patched-ultralytics-repo>"
```

5. Check the environment:

```bash
python tools/check_env.py
```

6. After applying the patch, run the focused tests from this release directory:

```bash
export PYTHONPATH=$ULTRALYTICS_PATH:$PYTHONPATH
python -m pytest tests
```

On Windows PowerShell:

```powershell
$env:PYTHONPATH="$env:ULTRALYTICS_PATH;$env:PYTHONPATH"
python -m pytest tests
```

## Example Commands

Standard training:

```bash
python -m mg_afss.run_mg_afss_train \
  --dataset nwpu_vhr10_full \
  --data dataset/nwpu_vhr10_full/data.yaml \
  --task detect \
  --model-size n \
  --train-mode scratch \
  --method off \
  --epochs 80 \
  --seed 42 \
  --device 0 \
  --batch 32 \
  --imgsz 1024 \
  --run-id NWPU_y11n_SCR80_OFF_s42
```

Fixed AFSS:

```bash
python -m mg_afss.run_mg_afss_train \
  --dataset nwpu_vhr10_full \
  --data dataset/nwpu_vhr10_full/data.yaml \
  --task detect \
  --model-size n \
  --train-mode scratch \
  --method afss \
  --warmup 20 \
  --epochs 80 \
  --seed 42 \
  --device 0 \
  --batch 32 \
  --imgsz 1024 \
  --run-id NWPU_y11n_SCR80_AFSSw20_s42
```

MG-AFSS:

```bash
python -m mg_afss.run_mg_afss_train \
  --dataset nwpu_vhr10_full \
  --data dataset/nwpu_vhr10_full/data.yaml \
  --task detect \
  --model-size n \
  --train-mode scratch \
  --method mg_afss \
  --mg-fit-min-points 5 \
  --moderate-ratio 0.40 \
  --epochs 80 \
  --seed 42 \
  --device 0 \
  --batch 32 \
  --imgsz 1024 \
  --run-id NWPU_y11n_SCR80_MGAFSS_s42
```

OBB experiments use `--task obb` and an OBB model/config.

## Reproducibility Notes

- The raw datasets must be obtained from their original providers.
- The paper compares methods under matched dataset, initialization, epoch budget, image size, batch size, seed, and runtime conditions.
- `summary.json`, `results.csv`, `afss_stats.json`, and `v3_log.json` are the main run-level evidence files generated by the training entry. The `v3_*` field names are retained for compatibility with historical experiment records; they correspond to MG-AFSS in the paper.

See `docs/DATASETS.md` and `docs/REPRODUCE.md` for details.
